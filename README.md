# AI Commercial Chatbot

AI-ассистент для подбора товаров с встраиваемым виджетом чата.

Проект помогает посетителю сайта подобрать товар через чат.

Он показывает виджет на сайте, принимает сообщение пользователя, ищет подходящие товары и показывает ответ в чате. Для просмотра работы есть демо-страница и панель с историей запросов.

## Что умеет проект

- Показывает чат-виджет на сайте.
- Отвечает на вопросы о товарах.
- Ищет товары в каталоге.
- Сохраняет историю запросов к модели.
- Показывает историю запросов в Streamlit.

## Два флоу

Бот работает в двух режимах:

1. Подбор товаров под задачу (`task`).
2. Ответы на вопросы о товарах (`consultation`).

### 1. Подбор товаров

Что делает:

- понимает задачу пользователя;
- может задать уточняющий вопрос;
- подбирает нужные товары;
- показывает подходящие позиции из каталога.

Пример:

1. Пользователь пишет: `Нужно поклеить обои в комнате`.
2. Бот понимает, что это задача на подбор товаров.
3. Если нужно, бот задаёт уточнение.
4. Потом показывает, что может понадобиться: обои, клей, грунтовка, валик, шпатель.

### 2. Ответы на вопросы

Что делает:

- понимает, что пользователь спрашивает про товар;
- отвечает коротко и по делу;
- при необходимости показывает товары из каталога.

Пример:

1. Пользователь пишет: `Что лучше для стен: валик или кисть?`
2. Бот понимает, что это вопрос-сравнение.
3. Отвечает, чем отличаются варианты и что выбрать в зависимости от задачи.
4. Если товары есть в магазине, показывает их карточки ниже ответа.

## Что где лежит

### `back/`

- Код для анализа запросов, поиска товаров и работы с моделью - здесь лежит логика, которая решает, что ответить пользователю.
- Панель Streamlit - показывает историю запросов и логи.
- Каталог товаров - используется для поиска подходящих товаров.

### `widget/`

- Сервер виджета и API чата - отдаёт страницу чата и обрабатывает сообщения.
- Статические файлы виджета - это `loader.js`, `chat.js` и стили.
- Шаблон чата - HTML-страница, внутри которой живёт чат.
- Настройки доменов - определяют, где виджет можно открывать.

## Как запустить в Docker

### 1. Скопируйте `.env`

```bash
cp .env.example .env
```

Вставьте туда свой ключ Mistral. Если ключа нет, попросите его в `tg: madvas`.

```env
MISTRAL_API_KEY=your_api_key_here
```

### 2. Соберите и запустите контейнер

```bash
docker compose up -d --build
```

### 3. Откройте ссылки

- `http://localhost:8001` - демо-страница с виджетом.
- `http://localhost:8501` - панель с логами и запросами.
